# 如何帮助和参与贡献

参与项目贡献是一个难度不小的工作，尽管如此，我们整理了不同难度的工作内容鼓励大家参与其中.

## 简单任务

1. 阅读说明书和wiki文档，修复文档错误和你发现不清楚的地方，并提交到Pull Requests.
2. 在你感兴趣的领域制作ezgal的相关教程、介绍、节目，或者帮助别人一起，如果是公开的，欢迎提交到`cryingnights7v@gmail.com`.
3. 尝试跑起来你的第一个ezgal框架项目，过程中出现任何问题欢迎反馈到issues.

## 中等任务

1. 参与讨论ezgal解释器的语法设计等相关事宜，欢迎分享大家的独到简介. 
2. 试着修复一些现有问题，比如issues中的报告，制作属于自己的Pull Requests.
3. 分享使用ezgal开发的游戏，我们欢迎各种优秀的作品，优秀的作品可以通过我们的通道进行展示曝光.

## 高级任务

1. 参与构建ezgal游戏框架的开发生态.
2. 联系我们`cryingnights7v@gmail.com`，参与项目安全相关维护工作.
3. 试着让ezgal变得更好，试着参与我们的编辑器开发相关工作.

# 标签说明

我们按需求定义了4组标签：

* fix: 项目需要修正
* new: 项目新增功能
* bug: 项目存在错误
* cve: 存在危害级别错误

按系统定义了3组标签：
* os:android
* os:Linux
* os:windows

便于大家在讨论和贡献中筛选自己擅长的工作.
